第二期 Spec 补充稿
### 版本定位
第二期目标是把当前“架构已搭好、部分功能已落地、部分 UI/持久化/搜索仍处于草稿或半接入状态”的阅读器，推进为一个可持续使用的桌面应用版本。

第一期已完成的基础包括：

+ 统一领域模型与状态模型
+ 主题系统与基础阅读链路
+ EPUB/TXT 基础解析与阅读
+ 部分 AppShell / panels / storage 模块骨架
+ 第二期不再以“新增结构”为主，而以“主路径收口、功能闭环、体验统一”为主。

### 第二期总目标
   + 正式切换到 AppShell + Panels + Widgets 主 UI 路径
   + 打通设置、阅读进度、书签、最近阅读的本地持久化
   + 补齐搜索、书签、快捷键、错误反馈等高频功能闭环
   + 提升 EPUB/TXT 的目录、元信息和章节切分质量
   + 使用前端页面 skill 统一梳理页面层次、状态页和视觉一致性
### 范围定义
   本期范围内：

+ src/ui/shell.rs 成为主入口 UI
+ TopBar / LeftSidebar / ReaderView / StatusBar / SearchPanel / SettingsPanel 收口
storage 层真实接入业务流程
+ 阅读工作流闭环：打开书、恢复进度、跳章、搜索、书签、最近阅读
+ EPUB 元信息与目录增强
+ TXT 自动切章
+ 页面级体验打磨和状态页统一

本期不做：

+ 云同步
+ 登录账号
+ PDF/MOBI 正式支持
+ 批注系统
+ WebView/Tauri/Slint 迁移
+ 大规模重构 parser 输出接口 

### 第二期关键设计决策
   当前运行入口统一切换到 AppShell，结束旧 UI 与新 UI 双轨并存状态。
   第一优先级是“闭环现有模块”，不是继续新增更多草稿模块。
   storage/* 不再只是工具模块，必须进入真实启动、打开、切章、退出流程。
   搜索、书签、最近阅读、设置的 UI 必须与状态层和存储层双向打通。
   前端页面 skill 用于输出页面结构与视觉规范，不替代 Rust UI 实现本身。
### 功能交付内容
#### UI 主壳层收口
   + AppShell 负责四类主状态：EmptyLibrary / LoadingBook / Reader / Error
   + TopBar 负责高频操作入口：打开书籍、最近、搜索、书签、设置、翻章
   + LeftSidebar 负责标签页切换：目录 / 书签 / 最近阅读
   + ReaderView 负责正文排版、章节头部、搜索高亮、滚动状态
   + StatusBar 负责阅读进度、章节位置、状态消息
   + 旧版 toolbar.rs / toc.rs / content.rs 不再作为长期主路径保留
#### 持久化与阅读行为
   + 启动时加载 settings.json
   + 启动时加载最近阅读列表
   + 打开书籍成功后读取该书 progress 与 bookmarks
   + 切章、关闭窗口、失焦时保存阅读进度
   + 修改设置后保存设置
   + 书签变更后立即保存
   + 最近阅读打开后更新排序与去重
   + 路径失效的最近阅读项要支持标记和清理
#### 搜索与书签闭环
   + 当前章节搜索
   + 全书搜索
   + 搜索结果列表展示
   + 搜索结果点击跳转
   + 正文命中高亮
   + 添加书签
   + 删除书签
   + 点击书签跳转到对应章节和段落
#### 解析增强
   + EPUB 元信息提取：title / creator / language / identifier / cover
   + EPUB 标准目录提取：优先 nav.xhtml，回退 ncx
   + TXT 自动切章：支持常见中文“第X章/节/卷”
   + 解析告警结构化：空章节、缺失目录、异常 href 等
   + 当前 epub.rs 的边界问题和告警项清理
### 页面与体验要求
   + 空状态页、错误页、加载页必须统一风格。
   + 正常阅读态中，正文仍然是视觉重心，工具区不应喧宾夺主。
   + 设置面板按分组组织：
     + 主题
     + 字体与字号
     + 行距与段距
     + 正文宽度与侧边距
   + 行为设置
     + 搜索面板必须支持“当前章节 / 全书”切换。
     + 最近阅读和书签的空状态要有完整文案与视觉反馈。
   + 使用前端页面 skill 输出页面基线稿，覆盖：
     + Reader 主界面
     + Search 展开态
     + Settings 展开态
     + Empty / Loading / Error / No Result / No Bookmark / No Recent
### 实施顺序
   + 收口 AppShell 主路径，切换正式运行入口
   + 接通 settings_store / progress_store / bookmark_store / recent_store
   + 打通书签、最近阅读、设置面板的真实行为
   + 完成搜索面板和搜索跳转
   + 增强 EPUB/TXT 解析质量
   + 进行页面级体验统一和回归测试
### 验收标准
   + 应用只保留一条正式 UI 主路径。
   + 重启应用后可以恢复设置和阅读进度。
   + 目录、书签、最近阅读三个侧栏标签都可真实使用。
   + 搜索支持结果展示、点击跳转和正文高亮。
   + EPUB 可以显示更准确的标题、目录和基础元信息。
   + TXT 支持自动切章，不再始终作为单块全文处理。
   + 空状态、加载状态、错误状态和正常阅读状态都完整且风格一致。
   + cargo test 通过，关键交互具备回归测试覆盖。
### 第二期完成标志
   当以下条件同时成立时，可视为第二期完成：

+ 用户可以长期使用该阅读器完成本地阅读
+ 关闭再打开不会丢失主要阅读状态
+ 新 UI 壳层成为唯一主路径
+ 搜索、书签、最近阅读和设置不是占位功能，而是完整闭环
+ 页面体验达到“产品级可用”，不再只是工程结构演示
